\title{Formalizing a Prime-Generated Version of Nagata's Factoriality Theorem in Lean}
\author{Arthur Freitas Ramos}
\date{}

\begin{abstract}
We formalize in Lean 4 / Mathlib a prime-generated version of Nagata's
factoriality theorem for noetherian integral domains. The development isolates
the localization lemmas needed to transfer irreducibility and primality between
a domain and a localization at a submonoid generated by prime elements, then
packages these ingredients into a reusable Nagata theorem. As a concrete
application, we derive a polynomial unique factorization result from a Laurent
polynomial localization hypothesis, showing that the formalized theorem performs
genuine mathematical work beyond thin wrappers around existing library
instances.
\end{abstract}

\section{Introduction}

Nagata's factoriality theorem is a standard tool in commutative algebra for
recovering unique factorization in a base ring from unique factorization in a
localization. It is therefore a natural candidate for formalization: the result
connects localization, irreducibility, primality, and noetherianity, and it
supports downstream arguments that would otherwise need to be reproved in a
less conceptual way.

This repository contributes a Lean formalization centered on the files
\texttt{NagataFactoriality/Nagata/Lemmas.lean},
\texttt{NagataFactoriality/Nagata/Theorem.lean}, and
\texttt{NagataFactoriality/Applications/Laurent.lean}. Early in the project,
the public theorem used the stronger assumption that every denominator is
already prime or a unit. For a submonoid, that hypothesis is too restrictive and
collapses the result to a degenerate special case. The present development
replaces it with a genuine prime-generated condition and rebuilds the
localization machinery around that corrected statement.

From a publication standpoint, the point is not merely to restate a classical
theorem in a proof assistant. The formal development exposes which pieces of the
argument are genuinely reusable library content, which parts are theorem-level
 mathematics, and how localization-heavy commutative algebra can be organized in
 Lean without hiding the key proof obligations behind opaque automation. That is
 exactly the sort of proof-engineering story that fits an ITP/FM-style paper.

\paragraph{Contributions.}
At the current repository state, the formal contribution is threefold.
\begin{enumerate}
  \item A prime-generated Nagata-style factoriality theorem for noetherian
  domains.
  \item A reusable localization API transferring divisibility, irreducibility,
  and primality information between a domain and its localization.
  \item A theorem-driven application showing how a Laurent-polynomial
  localization hypothesis yields a polynomial unique factorization result via
  the formalized Nagata theorem.
\end{enumerate}

\section{Formal statement and artifact structure}

The main public theorem is \texttt{nagata\_theorem} in
\texttt{NagataFactoriality/Nagata/Theorem.lean}. Its central hypothesis is the
predicate \texttt{PrimeGenerated S}, asserting that every element of the
multiplicative set admits a factorization into prime elements that remain inside
\texttt{S}. This is the right formal abstraction for the localization argument:
the prime factors of denominators must themselves become units after
localization.

The repository also preserves the earlier prime-or-unit statement as the
restricted compatibility theorem
\texttt{nagata\_theorem\_of\_prime\_or\_unit}. This is useful for comparison and
for explaining how the corrected theorem strictly generalizes the earlier
version without discarding the simpler interface entirely.

\section{Proof architecture}

The proof is organized around a small chain of reusable ingredients.
\begin{enumerate}
  \item \textbf{Prime-generated submonoid API.} The file
  \texttt{NagataFactoriality/Nagata/Lemmas.lean} introduces
  \texttt{PrimeGenerated} and basic factorization lemmas for products of prime
  denominators.
  \item \textbf{Divisibility transfer lemmas.} Intermediate results show that an
  irreducible element dividing an element of a prime-generated submonoid is
  prime, and that divisibility information can be lifted back from the
  localization.
  \item \textbf{Localization-to-base-ring bridge.} The key theorem
  \texttt{nagata\_key\_lemma\_primeGenerated} packages the previous lemmas into
  the exact primality transfer needed by the final UFD argument.
  \item \textbf{Final UFD packaging.} The public theorem then combines the key
  lemma with the existing UFD infrastructure to conclude unique factorization in
  the base ring.
\end{enumerate}

This architecture matters for publication because the reusable content is not
just the final statement. Several of the intermediate localization lemmas are
plausible candidates for later Mathlib-oriented cleanup or upstreaming.

\section{Application}

The current application lives in
\texttt{NagataFactoriality/Applications/Laurent.lean}. It works with the
submonoid of powers of $X$ in a polynomial ring, proves that this multiplicative
set is prime-generated, transports unique factorization from a Laurent
polynomial localization, and then applies the corrected Nagata theorem to obtain
a polynomial UFD result. This gives the repository a concrete example in which
the main theorem performs essential mathematical work.

\section{Related work and novelty positioning}

The classical mathematical background comes from Nagata's treatment of
localization and factoriality \cite{nagata1962}, together with the surrounding
 UFD literature in Samuel and Matsumura \cite{samuel1964,matsumura1989}. On the
 proof-assistant side, the relevant Lean baseline is mathlib, a large
 community-maintained library that already provides the general commutative
 algebra and localization infrastructure on which this repository builds
 \cite{mathlib}. Our contribution is therefore not the invention of localization
 or UFD machinery in Lean, but the formal packaging of a specific Nagata-style
 theorem and the proof architecture needed to make it reusable.

For nearby formal work in other systems, Coq's Mathematical Components library
 offers a mature algebraic environment with strong support for divisibility,
 polynomial algebra, and the style of reusable interfaces needed for
 mechanized algebra \cite{mathcomp}. In Isabelle/HOL, Bordg's AFP entry
 \emph{The Localization of a Commutative Ring} provides a direct formalization of
 localization as a reusable commutative-algebra construction
 \cite{bordg-localization}, while Divas{\'o}n, Joosten, Thiemann, and Yamada
 formalize a verified Berlekamp--Zassenhaus factorization algorithm in
 Isabelle/HOL, showing that substantial factorization infrastructure has also
 been developed there \cite{divason-berlekamp}.

\paragraph{Novelty claim.}
The claim of novelty should be stated carefully. Based on current public library
 documentation and repository searches across Lean/mathlib, Coq/MathComp, and
 Isabelle/AFP, we are not aware of a public formalization of Nagata's
 factoriality theorem itself. That is a claim about the present public state of
 the libraries, not a proof that no unpublished or in-progress formalization
 exists. The strongest responsible phrasing is therefore: this repository appears
 to contribute a new formalization of Nagata's factoriality theorem, while
 building on already mature formal algebra infrastructure.

\section{Prospective library contributions}

Not every theorem in the repository should be marketed as a potential upstream
 contribution. A reasonable split is as follows.
\begin{enumerate}
  \item \textbf{Plausibly upstreamable.} The lightweight localization helpers in
  \texttt{NagataFactoriality/Localization/*.lean} and the generic submonoid
  lemmas about zero and nonzero denominators are reusable beyond the present
  theorem and are good candidates for later Mathlib-oriented cleanup.
  \item \textbf{Potentially upstreamable after refactoring.}
  The \texttt{PrimeGenerated} interface and some of its generic factorization
  lemmas are mathematically natural, but should only move upstream if their API
  stabilizes and if their current proofs can be reorganized in a way that fits
  Mathlib's preferred abstraction boundaries.
  \item \textbf{Paper-level contribution.} The final Nagata theorem and its
  theorem-specific proof packaging should presently be treated as the core
  contribution of this repository rather than as assumed near-term upstreaming
  targets.
\end{enumerate}

\section{Reproducibility}

The repository targets Lean 4.24.0 and builds with Lake. The core reproducible
commands are:
\begin{verbatim}
lake exe cache get
lake build
\end{verbatim}
The repository also includes CI and citation metadata, so the artifact is now
both buildable and citable.

\paragraph{Release packaging.}
For paper submission, the recommended artifact is a tagged source release that
 freezes \texttt{lean-toolchain}, \texttt{lake-manifest.json}, the Lean sources,
 \texttt{paper/main.tex}, and \texttt{CITATION.cff}. The paper should cite that
 tagged release (or, at minimum, a commit hash) rather than an unspecified moving
 branch. If the venue accepts archival artifacts, the same release can be
 mirrored to a DOI-minting archive such as Zenodo.

\section{Limitations and future work}

The remaining publication work is mostly expository rather than foundational.
The theorem and its application are formalized and build cleanly, but the paper
will still benefit from venue-specific prose polishing, bibliography-format
cleanup, and a final release cut at submission time.
